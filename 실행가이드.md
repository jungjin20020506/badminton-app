# 실행 가이드

## ✅ 설치할 것이 없습니다

**Python을 따로 깔 필요 없습니다.** 프로그램 폴더 안 `runtime` 폴더에 실행에 필요한 것이
모두 들어 있어서, **폴더만 복사하면 어느 PC에서든 바로 실행**됩니다.

## 파일은 이것만 보시면 됩니다

| 파일 | 언제 쓰나 |
|---|---|
| **내PC에서_실행하기.bat** | **사내 서버에 둔 프로그램을 쓸 때** — 이 파일만 내 PC로 복사해 실행 |
| **실행하기.bat** | 프로그램 시작 (창 없이 조용히 실행됩니다) |
| **종료하기.bat** | 프로그램 종료 |
| **팀서버_시작.bat** | 팀이 같이 쓸 때 (한 대만 실행) |
| **진단하기.bat** | 실행이 안 될 때 원인 찾기 |

> ⚠️ 폴더를 복사할 때는 **폴더 전체를 통째로** 복사하세요.
> `runtime` 폴더가 빠지면 실행되지 않습니다.

---

## 1. 혼자 쓰기

1. 프로그램 폴더를 내 PC로 복사
2. `실행하기.bat` **더블클릭**
3. 잠시 뒤 브라우저가 자동으로 열립니다 — 끝입니다

> 검은 터미널 창은 뜨지 않습니다. 프로그램은 뒤에서 조용히 돌아갑니다.
> 끄고 싶으면 `종료하기.bat` 을 더블클릭하세요. (안 꺼도 PC를 끄면 같이 종료됩니다)

### 주소가 `http://localhost:8000` 으로 뜨는 이유
이 프로그램은 **내 PC 안에서 도는 작은 웹 프로그램**이라 브라우저로 여는 구조입니다.
`localhost`(=127.0.0.1)는 "내 컴퓨터"를 뜻하는 주소로, 인터넷에 나가지 않고
**내 PC 안에서만** 연결됩니다. 즉 외부에서는 접근할 수 없어 안전합니다.

---

## 2. 팀이 같이 쓰기 (권장)

**한 대의 PC만 서버로 켜두고, 나머지는 브라우저로 접속**합니다.
데이터가 한 곳에만 쌓여 흩어지지 않고, 서로 덮어쓸 걱정도 없습니다.

### 서버로 쓸 PC (1대)
1. `팀서버_시작.bat` **더블클릭**
2. 처음 실행 시 Windows 방화벽 창이 뜨면 → **[액세스 허용]** 클릭
3. 화면에 표시되는 주소를 팀원에게 알려주세요
   ```
   http://192.168.0.15:8000     ← 이런 형태로 표시됩니다
   ```
4. 이 창은 **계속 켜 두세요** (닫으면 팀원도 사용 불가)

### 나머지 팀원 PC
- 브라우저 주소창에 위 주소만 입력하면 끝입니다
- **아무것도 설치하거나 복사할 필요가 없습니다**

> 💡 서버 PC는 자주 켜져 있는 PC로 정하세요. 팀원은 즐겨찾기에 추가해 두면 편합니다.

---

## 3. 사내 서버(공유폴더)에 두고 쓰기 ★

프로그램을 사내 서버에 올려두면 팀원 모두가 최신본을 쓸 수 있습니다.

> 예) `X:\연구소문서\출하 관련 자료\15. 품질 AI 프로그램\badminton-app-main`

### 처음 한 번만 (30초)

1. 서버 폴더에서 **`내PC에서_실행하기.bat`** 파일 **하나만** 복사 (Ctrl+C)
2. **내 PC 바탕화면**에 붙여넣기 (Ctrl+V)
3. 바탕화면의 그 파일을 **더블클릭**

끝입니다. 이후로는 **바탕화면 아이콘만 더블클릭**하면 되고,
실행할 때마다 서버의 최신본을 자동으로 받아옵니다.

### ⚠ 서버 폴더의 `실행하기.bat` 을 직접 누르면 안 되나요

**차단됩니다.** 이런 창이 뜹니다:

> 스마트 앱 컨트롤이 안전하지 않을 수 있는 파일을 차단했습니다.
> 이 파일은 **인터넷에서** 이 형식의 파일이 위험할 수 있으므로 차단되었습니다.

사내 서버가 **IP 주소**(`\\192.168.123.5\...`)로 연결돼 있어서
Windows 가 이 위치를 **"인터넷 영역"** 으로 취급하기 때문입니다.
파일이 위험해서가 아니라, **위치** 때문에 막는 것입니다.

그래서 **처음 클릭하는 파일이 내 PC에 있어야** 하고,
그 역할을 하는 것이 `내PC에서_실행하기.bat` 입니다.

> 💡 파일을 **복사**하는 것은 "실행"이 아니라서 차단되지 않습니다.
> 복사한 파일은 내 PC 파일이 되므로 그냥 실행됩니다.

### 왜 내 PC로 받아서 실행하나요

- **스마트 앱 컨트롤**은 네트워크(인터넷 영역) 폴더의 실행 파일을 막습니다.
- **데이터 파일(SQLite)을 공유 폴더에 두고 여러 명이 동시에 쓰면 DB가 깨집니다.**
  프로그램만 서버에서 공유하고, 데이터는 각 PC에 안전하게 보관합니다.

### 전산팀이 있다면 (선택)

사내 파일서버를 **로컬 인트라넷 영역**으로 등록하면 이 차단이 아예 사라집니다.
Windows 설정 → **인터넷 옵션 → 보안 → 로컬 인트라넷 → 사이트 → 고급** 에서
`\\192.168.123.5` 를 추가하면 됩니다. (보안 설정이라 담당자 승인 후 진행하세요)

### 데이터는 어떻게 되나요

- 서버 폴더에 기존 데이터가 있으면, **처음 실행할 때 한 번 내 PC로 가져옵니다.**
  (서버 원본은 그대로 두고 복사만 합니다)
- 그 뒤로는 **각자 PC에 따로** 쌓입니다.
- 출하이슈사항의 **원본은 항상 Z: 서버**이므로, 각자 동기화를 돌리면 같은 내용을 받습니다.
- **한 벌의 데이터를 팀이 함께** 쓰려면 위 **2번(팀서버)** 방식을 사용하세요.

### Z: 서버 연동은 그대로 됩니다

서버에서 실행하든 내 PC에서 실행하든 **Z: 출하이슈사항 동기화·사진·엑셀 미리보기는 동일하게 동작**합니다.
(Z: 드라이브는 각 PC에 연결된 것을 사용합니다. 드라이브 문자가 다르면 이슈관리 화면의 **서버 경로 설정**에서 바꿔 주세요)

## 자주 있는 문제

| 증상 | 해결 |
|---|---|
| "runtime 폴더가 없습니다" | 폴더를 복사할 때 일부만 복사된 경우입니다. **원본 폴더를 통째로** 다시 복사하세요 |
| 창이 뜨자마자 닫힘 | 위와 같습니다. 폴더 전체를 다시 복사해 주세요 |
| 팀원이 접속 안 됨 | 서버 PC 방화벽에서 **[액세스 허용]** 을 눌렀는지, 같은 사내망인지 확인 |
| 브라우저가 안 열림 | 주소창에 `http://127.0.0.1:8000` 직접 입력 |
| 화면이 예전 그대로 | 검은 창을 닫고 다시 실행 → 브라우저에서 **F5** |
| 서버 동기화가 안 됨 | 이슈관리 화면에 **서버 경로 입력칸**이 자동으로 나타납니다. 내 PC에서 사내 서버가 어떤 드라이브인지 확인해(예: `Z:\` 또는 `Y:\`) 입력 후 저장하세요. `\\서버이름\공유폴더` 형태도 됩니다 |
| **스마트 앱 컨트롤이 차단함** ("이 파일은 인터넷에서...") | 서버 폴더의 `.bat` 을 직접 누르면 차단됩니다. **`내PC에서_실행하기.bat` 을 내 PC(바탕화면)로 복사한 뒤 거기서 실행**하세요 → 위 **3번** 참고 |
| 백신이 차단함 | `runtime` 폴더를 예외 등록하거나 전산팀에 문의하세요 (python.org 공식 배포본이며 서명이 확인됩니다) |
| 원인을 모르겠음 | **`진단하기.bat`** 을 더블클릭하면 실행 위치·런타임·Z: 연결·포트 상태를 한 번에 점검해 줍니다 |

## 데이터는 어디에 있나요

- **`data\quality.db`** 파일 하나입니다
  - 내 PC 폴더에서 실행 → 그 폴더의 `data\quality.db`
  - **사내 서버 폴더에서 실행 → `%LOCALAPPDATA%\KNK출하검증\data\quality.db`**
    (공유 폴더에 DB를 두면 동시 사용 시 깨지므로 내 PC에 보관합니다)
  - 지금 위치는 프로그램 화면 하단 안내에서도 확인할 수 있습니다
- **프로그램 시작 시 자동 백업**됩니다 (`data\backups\`, 최근 14개 보관)
- 옮기거나 백업하려면 `data` 폴더를 통째로 복사하면 됩니다
- ⚠️ 고객사 이슈 데이터가 들어있습니다. **외부(인터넷·개인 클라우드)로 유출되지 않게** 주의하세요
